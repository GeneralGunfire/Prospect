# Plan 00-01: Vite + React + TypeScript + Tailwind v4 Scaffold

## Objective

Bootstrap the project with the locked stack: React 19 + Vite 8 + TypeScript + Tailwind CSS v4 (Vite plugin) + React Router v7 (library mode). Stub all routes. Configure Vercel cpt1 deployment. The output is a running dev server and a deployable shell under 50KB gzipped.

## must_haves

- App shell builds and serves without errors
- `vercel.json` sets `"regions": ["cpt1"]` before any deploy
- Tailwind v4 uses `@tailwindcss/vite` plugin — NOT PostCSS
- React Router v7 in library mode with all routes stubbed
- Initial gzipped bundle is under 50KB (verify with `vite build --report`)

### Task 5: Configure Vercel deployment with Cape Town (cpt1) region

<read_first>
- vercel.json (check if exists; create if not)
</read_first>

<action>
Create `vercel.json` in the project root:

```json
{
  "regions": ["cpt1"],
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

The `rewrites` rule is required for React Router client-side routing — without it, direct URL navigation (e.g. `/careers/123`) returns a 404 from Vercel.

**CRITICAL:** `"regions": ["cpt1"]` MUST be present before first deploy. Vercel defaults to `iad1` (Washington DC). Deploying to the wrong region and switching later is possible but forces cache invalidation.
</action>

<acceptance_criteria>
- `vercel.json` exists in project root
- `vercel.json` contains `"regions": ["cpt1"]`
- `vercel.json` contains a `rewrites` rule routing `(.*)` to `/index.html`
- `vercel.json` is valid JSON (no syntax errors)
</acceptance_criteria>


Common fixes if over budget:

- Verify Supabase JS is NOT imported anywhere in the initial bundle — it must be dynamically imported only
- Verify React Router is code-split (the `manualChunks` config in vite.config.ts handles this)

- `vendor` chunk (react + react-dom): ~45KB gzipped
- `router` chunk: ~25KB gzipped
- `index` chunk (app shell): ~5KB gzipped
- CSS: ~3KB gzipped
- **Total on first load: ~50KB** (vendor + router are split but both load on first visit — optimize in Phase 6)

Note: Achieving strict <50KB on the initial JS may require deferring the router chunk. In Phase 0, achieve <80KB and log this as a Phase 6 task. The 50KB requirement (PLAT-01) applies to production after PWA optimization in Phase 6.
