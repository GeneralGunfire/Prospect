# Plan 02-01: TypeScript Schema Design + JSON Schema Audit

## Objective

Finalise all JSON schemas and TypeScript interfaces before any display component is built. Audit existing JSON files in the repo against the required schemas and produce canonical, migration-ready versions. This is the highest-leverage design decision — schema changes after UI is built are the most expensive rework.

## must_haves

- `src/data/types.ts` has zero `any` types
- `subjects_required[]` array with `{ subject, min_level, mandatory }` shape exists on both `CareerIndexEntry` and `Career`
- Every bursary record has a non-empty `url` field
- TypeScript interfaces match JSON structure exactly — no silent coercions
- All existing career JSONs in repo are audited and note what fields are missing

4. If `demand_level` is missing, add `"medium"` as placeholder
5. Remove any fields not in `CareerIndexEntry` (keep data clean)

Ensure the file has entries for at least 2 complete careers with full `subjects_required` arrays.

Write the updated file back to `public/data/_master-index.json`.
</action>
