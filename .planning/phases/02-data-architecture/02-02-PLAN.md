# Plan 02-02: Bursary JSON Audit + Job Gaps Data + Sample Careers

## Objective

Audit the existing 245-bursary JSON for schema compliance (every record must have a valid `url`), create the province job demand matrix (`job-gaps.json`), and add 3 more full career JSONs (software developer, electrician, plumber) to give Phase 3 enough data to demonstrate the filtering UI.

## must_haves

- Every bursary record has a non-empty `url` pointing to an official source
- `job-gaps.json` covers all 9 provinces with top careers and key industries
- At least 5 full career JSONs exist across `university/` and `tvet/` subfolders
- `_master-index.json` includes all 5 careers with correct `pathway` fields

## Tasks

### Task 1: Audit bursaries.json for URL compliance

<read_first>
- public/data/bursaries.json (existing bursary data — check current location: may be in data/ not public/data/)
- src/data/types.ts (Bursary interface)
</read_first>

<action>
Read the existing `bursaries.json`. Check the schema:

1. Find all records where `url` is missing, empty, or `null` — log them
2. For records missing a URL, add a placeholder: `"url": "https://www.nsfas.org.za"` (general NSFAS) or the provider's website root
3. Ensure every record has: `id`, `name`, `provider`, `fields` (array), `provinces` (array), `income_max`, `sa_citizen_required`, `first_time_student`, `url`
4. Add missing fields with sensible defaults:
   - `aps_minimum`: `null` if not applicable
   - `closing_date`: `null` if rolling
   - `value`: `null` if not specified
   - `income_max`: `null` if no income restriction
5. Write the corrected file back to `public/data/bursaries.json`

Key bursaries to ensure are present with correct URLs:
- NSFAS → `https://www.nsfas.org.za`
- Sasol Bursary → `https://www.sasol.com/careers/bursaries`
- SANRAL → `https://www.sanral.co.za/bursaries`
- Eskom → `https://www.eskom.co.za/bursaries`
- Transnet → `https://www.transnet.net/bursaries`
- Anglo American → `https://www.angloamerican.com/careers/bursaries`
- FirstRand (FNB/RMB) → `https://www.firstrand.co.za/bursaries`
- Standard Bank → `https://www.standardbank.co.za/bursaries`
</action>

<acceptance_criteria>
- `public/data/bursaries.json` is valid JSON
- `node -e "JSON.parse(require('fs').readFileSync('public/data/bursaries.json','utf8')).filter(b => !b.url).length"` returns `0`
- `node -e "console.log(JSON.parse(require('fs').readFileSync('public/data/bursaries.json','utf8')).length)"` returns at least `200`
- Every bursary entry has a `url` field with a non-empty string value
</acceptance_criteria>
