Prospect — Section 1
01
Overview
What Prospect Is and Why It Exists

Prospect is a free, mobile-first web platform for South African high school students in Grades 8 to 12. It gives students access to career guidance, bursary information, a curriculum-focused AI assistant, and live data on the infrastructure issues — load shedding and water supply — that directly affect how and when they can study.

The platform started as a career and subject guidance tool. Research into the actual problems South African learners face expanded it. Load shedding disrupts study sessions. High data costs mean students can't afford slow sites. Guidance counsellors are too scarce in public schools to reach every student who needs one. The platform grew to address each of these things directly — and every feature that couldn't justify itself against a real problem was removed before launch.

Two features that were prototyped but removed: a pothole reporting tool (not relevant to learners) and a community impact visualisation (data didn't exist in a usable form). What remained after that cut is what launched.

The Four Platform Pillars
Pillar 1
Careers and Guidance
Career explorer (400+ profiles), career quiz (RIASEC-style), career roadmap, grade 10 subject selector, TVET section with colleges, careers, funding, and requirements.
Pillar 2
Bursaries and Funding
245 verified bursary listings with full eligibility, coverage, deadline, and direct application link. Filterable by field, province, and open/closed status.
Pillar 3
School Assist
AI chat assistant powered by Groq (llama-3.3-70b-versatile), study library with 24 topics across 11 subjects, interactive algebra modules with progress tracking, academic calendar, matric exam countdown.
Pillar 4
Community and Life Context
Live load shedding tracker (Eskom data, updated every 30 minutes), water dashboard (provincial dam levels, daily), cost of living by city, civics guide, tax and budget explainer.
Who it is for

The primary user is a Grade 10–12 learner in a South African public school, likely accessing the platform on a smartphone on prepaid mobile data. The platform is also useful for Grade 8 and 9 learners who want early visibility into how subject choices affect career options. Everything that requires login is clearly marked — most features are accessible without creating an account.

Architecture and Tech Stack
System Architecture
Browser
React 19 SPA — all UI, routing, state, and data rendering happens client-side. No server-side rendering. The app loads once and navigates internally.
Routing
Custom page state in App.tsx — a page state variable holds the current view. ?page=X URL param enables direct links and Playwright test navigation. React.lazy() and Suspense wrap every page component for code-split loading.
Data
TypeScript data files — careers400Final.ts (assembled from 21 batch files across 11 sectors), bursaries.ts (245 entries with typed Requirement and Coverage interfaces), tvetColleges.ts, tvetCareers.ts, studyLibrary/index.ts. All imported directly, no API call for this data.
Auth
Supabase Auth — email/password sign-up and sign-in, session persistence via localStorage, auto session restore on page load. withAuth.tsx HOC wraps protected pages and redirects unauthenticated users. Test mode bypass: ?__test_mode=true skips all auth checks for Playwright.
Database
Supabase PostgreSQL — stores user profiles, forum posts, study progress (mastery tracking for algebra modules). supabaseSync.ts runs background sync between localStorage and Supabase so progress persists across devices.
AI
Groq API — llama-3.3-70b-versatile model via fetch to api.groq.com/openai/v1/chat/completions. API key stored as VITE_GROQ_API_KEY in Vercel environment variables. OpenAI-compatible request format. 10-message rolling context window per session.
Live Data
GitHub Actions scrapers (Python) — load shedding scraper runs every 30 minutes, water scraper runs daily at 07:00 SAST. Both commit JSON output to public/data/. Vercel auto-deploys on every commit, making scraped data live.
Hosting
Vercel — continuous deployment from GitHub main branch. Environment variables managed in Vercel dashboard. Free tier at launch.

Feature Detail
Careers Pillar — Explorer, Quiz, Roadmap, TVET
Career Explorer

The career explorer lives at the careers page and is built on top of careers400Final.ts — a TypeScript file that assembles 400+ career profiles from 21 batch import files covering sectors including engineering, healthcare, trades, business, digital, education, legal, finance, media, agriculture, and public services. Each profile is typed against the CareerFull interface defined in careersTypes.ts.

Each career entry contains: career title, sector, required subjects, qualification level, APS range, salary range (SA-specific), day-in-the-life description, qualification path, where to study, and a provincial job demand indicator. Students can search by name, filter by subject, sector, or qualification level, and click through to a full detail view without needing to be signed in.

Career Quiz

The quiz page (QuizPage.tsx) runs a multi-step questionnaire. Students select their current subjects and rate their interests. The scoring logic in quizScoringLogic.ts and quizService.ts calculates a fit score for each career and returns a ranked list of matches. Each result links directly to the career detail page. No sign-in required.

Career Roadmap

A visual step-by-step path from a student's current grade to a chosen career — showing required subjects, qualification milestones, and estimated time at each stage. Driven by data in demoLearningPath.ts and subjectCareerMapping.ts.

Grade 10 Subject Selector

Grade10SubjectSelectorPage.tsx lets a student select the subjects they are currently taking or considering. The platform immediately shows which careers those subjects open up and which they close off — making the consequence of subject choices visible before they're finalised.

TVET Section

The TVET section is a standalone sub-platform with five linked pages, all under src/pages/tvet/:

PageContent
TVETPageOverview hub — what TVET is and how it compares to university
TVETCareersPageTrade and vocational career listings from tvetCareers.ts
TVETCollegesPage26 TVET colleges from tvetColleges.ts — province, campuses, programmes
TVETFundingPageFunding options for TVET students (NSFAS, SETA, other)
TVETRequirementsPageAdmission requirements by programme and college

The TVET section deliberately presents vocational routes as a first-choice pathway alongside university options — not as a fallback. This is a design decision, not just a layout choice.

Province Job Demand Map

MapPage.tsx renders an interactive Leaflet map (using react-leaflet) showing job demand by province for different career sectors. Data is drawn from mapData.ts and updated via mapService.ts. Students can select a career or sector and see which provinces have high, medium, or low demand — useful for relocation planning around study or work.

Bursaries Pillar — Finder and Detail Pages
Data Structure

All bursary data lives in src/data/bursaries.ts, last verified March 2026. The file defines two key interfaces — Requirement and Coverage — and exports an array of 245 bursary objects. Each entry includes:

FieldType / Example
providerString — e.g. "Funza Lushaka", "NSFAS", "Sasol"
fieldString — field of study the bursary covers
provinceString or "National"
requirementsRequirement — minMarks, maxIncome, citizenship, studyLevel
coverageCoverage — tuition, accommodation, meals, books (booleans)
deadlineString — verified for 2024/2025 cycle
status"open" | "closed" | "unknown"
applicationLinkVerified HTTPS URL (primary source only)

Bursaries where deadlines could not be verified against a primary source are marked status: "unknown" rather than shown with potentially incorrect dates.

Bursary Search Page

BursariesPage.tsx loads the full 245-entry dataset (code-split into its own Vite chunk, so it only loads when this page is opened). Students can:

  • Search by bursary name, provider, or field of study
  • Filter by field of study
  • Filter by province
  • Filter by open/closed/unknown status

Each card in the results shows the provider, field, amount (where known), deadline, and eligibility summary. Clicking through opens BursaryDetailPage.tsx with the full profile including the Coverage breakdown (exactly what is and isn't funded), required documents, and the direct application link.

Why This Matters

Bursary information in South Africa is fragmented across dozens of company websites, government portals, and third-party aggregators — each with different levels of accuracy and currency. The 245 listings in Prospect were verified against primary sources (provider websites, government portals) before being included. Third-party aggregators were used only for discovery. Entries with unverifiable data were excluded or flagged rather than included with incorrect information.

School Assist — AI Chat and Study Library
AI School Assistant — How It Works

SchoolAssistChatPage.tsx handles the full chat interface. When a student sends a message, the following happens in sequence:

1
Predefined answer check
The message is checked against 5 hardcoded starter questions (subjects/APS, NSFAS, career guidance, university entry, study tips). If it matches exactly, the predefined answer renders immediately from local memory — no API call, under 50ms.
2
Groq API call
If no predefined match: a fetch POST is sent to api.groq.com/openai/v1/chat/completions with the system prompt, up to the last 10 messages as history, and the new user message. Model: llama-3.3-70b-versatile. Temperature: 0.7. Max tokens: 1024.
3
System prompt context
The system prompt instructs the model to act as a South African school guidance counsellor — focusing on matric subjects, APS scores, NSFAS, CAO applications, TVET pathways, and exam preparation. Formatted with bold and bullet points for mobile readability.
4
Image upload path
Students can attach an image (homework, exam paper) via the paperclip button. The file is converted to base64 and sent with the message. The model can read and respond to the image content. Image requests take approximately 14 seconds — noticeably slower than text, by design.
5
Topic chips
Six quick-tap chips (Subjects, APS Score, Funding, University, Study Tips, Career Quiz) appear above the input. Tapping one sends a preset question instantly — no typing required.
Study Library

StudyLibraryPage.tsx presents structured study material for 24 topics across 11 subjects, all sourced from src/data/studyLibrary/. Each topic file exports structured notes: headings, worked examples, key terms, and practice questions. Topics are CAPS-aligned — they follow the actual NSC curriculum structure.

Interactive Algebra Modules

Two fully interactive modules exist under src/pages/learning/Algebra/Grade10/Term1/: Linear Equations and Simultaneous Equations. Community and Life Context Features

The community pillar grew out of a research finding: South Africa's infrastructure realities — load shedding and water supply disruptions — directly reduce how much students can study. Knowing about a Stage 4 power cut in advance lets a student plan a study session around it. Not knowing means a lost evening. These are not novelty features — they address a real, measurable problem.

Load Shedding Tracker

LoadSheddingPage.tsx fetches data from public/data/loadshedding/latest.json — a file maintained by a Python scraper (backend/loadshedding_scraper.py) running via GitHub Actions every 30 minutes. The scraper queries Eskom's public API endpoint and commits updated JSON if the stage has changed. Vercel redeploys on every commit, making the data live within minutes of any stage change.

The page shows the current Eskom stage and gives students context on what that means for their area — so they can decide when to charge devices, download study materials, or plan offline study time.

Water Dashboard

WaterDashboardPage.tsx uses data from public/data/water/latest.json, populated by backend/water_scraper.py running daily at 07:00 SAST. The page — built across four sub-components (DamTrendsChart.tsx, PreparednessSection.tsx, ResearchSection.tsx, RestrictionLevelGuide.tsx) — shows provincial dam levels, current water restriction levels, and a trends chart. Students in water-affected areas (particularly in the Western Cape and parts of KwaZulu-Natal) can see real current conditions.

Cost of Living

CostOfLivingPage.tsx shows cost of living data by city — rent, food, and transport across Cape Town, Johannesburg, Durban, Pretoria, and Port Elizabeth. The primary purpose is to help students evaluate a bursary offer realistically: a bursary that covers tuition but not accommodation is worth less in Cape Town than in a smaller city. Data sourced from Numbeo and cross-referenced with StatsSA consumer price data.

Civics Guide

CivicsPage.tsx covers South African rights, government structure, and civic processes — the content that school doesn't always teach clearly or that students struggle to find consolidated in one place. Based directly on the Constitution of South Africa (1996), Chapter 2 (Bill of Rights) and the gov.za government structure explainer. Written in plain language for high school readers.

Tax and Budget Explainer

TaxBudgetPage.tsx explains how South African government tax and budget allocation works — what tax is, how it is collected, and how public money gets allocated to departments including education and health. Data drawn from the National Treasury budget documentation and updated to reflect the 2025/26 budget cycle.

Auth, Data Persistence, and Backend Services
Authentication Flow
1
Student lands on any page
App.tsx runs supabase.auth.getSession() on load. If a session exists in localStorage, the user is restored automatically without needing to log in again.
2
Accessing a protected page
withAuth.tsx wraps protected pages. If no session is found, the user is redirected to AuthPage. If ?__test_mode=true is present in the URL, auth is bypassed entirely — used by Playwright tests in CI.
3
Sign up / Sign in
AuthPage.tsx calls supabase.auth.signUp() or supabase.auth.signInWithPassword() from lib/auth.ts. On success, Supabase returns a session that is automatically stored in localStorage and picked up on subsequent loads.
4
Background sync
supabaseSync.ts runs in the background after sign-in, syncing any study progress that was saved to localStorage during an anonymous or previous session into the Supabase database so it persists across devices.

Service Layer

The src/services/ directory contains dedicated service modules that separate data-fetching and business logic from UI components:

  • loadshedding.ts — fetches and caches load shedding JSON
  • waterService.ts — fetches and caches water dashboard JSON
  • quizService.ts — handles quiz scoring and result sorting
  • bookmarkService.ts — saves and retrieves bookmarked careers/bursaries

  • storageService.ts — abstracts localStorage reads/writes
  • supabaseSync.ts — syncs local state to Supabase database
  • schoolAssistService.ts — fuzzy search using Levenshtein distance for topic lookup in the study library

User Experience
User Workflow — A Student's Full Session

This section walks through what a Grade 11 student actually does from the moment they open Prospect to the point where they have useful, actionable information about their future. It is not a hypothetical — it reflects the actual navigation paths the platform supports.

Scenario: "I don't know what to study after matric"
1
Arrives at the home page
The landing page (App.tsx) shows four pillar sections and a "How It Works" section. An animated hero draws attention to the career quiz as the starting point for undecided students. A 4-step tutorial dialog appears on first visit, walking through the platform's main features.
2
Takes the career quiz
QuizPage.tsx — no login required. The student selects their current subjects and rates their interest areas. Results appear as a ranked list of career matches. Each card shows the career name, sector, and fit score with a link to the full profile.
3
Opens a career detail
The career detail view shows everything: what the job involves, required subjects, APS range, salary range, where to study in SA, and what to be doing in each grade to get there. All sourced from the careers400Final.ts typed database.
4
Looks for bursaries for that career
Student navigates to Bursaries, filters by the relevant field of study and their province. Results show 245 listings narrowed to matching entries. Student opens a bursary detail, reads the coverage breakdown, and follows the direct application link.
5
Asks the AI assistant a follow-up
Student opens School Assist Chat and asks "Do I need Maths or Maths Literacy for engineering?" The question is not a predefined answer, so it hits Groq. Response arrives in under 2 seconds. The system prompt ensures the answer is SA-specific (NSC, APS context).
6
Checks the load shedding stage
Before starting an online study session, the student checks the Load Shedding page. Current Eskom stage is shown from the JSON file last updated by the GitHub Actions scraper. Student sees Stage 3 is active until 22:00 and plans accordingly.
Scenario: "I want to explore TVET instead of university"
1
Navigates to the TVET section
TVETPage.tsx explains what TVET is, how it compares to university pathways, and why it is a valid first choice — not a fallback. The nav sub-menu (TVETSubNav.tsx) provides direct links to careers, colleges, funding, and requirements.
2
Browses TVET colleges
TVETCollegesPage.tsx shows 26 public TVET colleges from tvetColleges.ts. Each entry shows the college name, province, campuses, and available programmes. The student filters by province to find colleges within reasonable distance.
3
Checks admission requirements
TVETRequirementsPage.tsx shows minimum entry requirements by programme — many TVET programmes require significantly lower NSC marks than university entry, and this is shown plainly so the student understands their options.
4
Finds TVET funding
TVETFundingPage.tsx covers NSFAS for TVET students, SETA funding programmes, and other bursaries applicable to vocational study — making clear that TVET students have access to the same major funding sources as university students.
5
Studies for an upcoming test
Student signs in to save progress. Opens Study Library, finds a Maths topic, works through it. If they open the Grade 10 Algebra linear equations module, their completion is tracked and written to Supabase — visible in their dashboard on next login.
Navigation Model

The app uses a custom page-state navigation model in App.tsx rather than URL-based routing. The navigate(page) function updates the page state, which triggers lazy loading of the relevant page component via React.lazy() and Suspense. The top nav bar (AppHeader.tsx) collapses on scroll using the usePillVisible hook to maximise screen space on mobile.

Testing, Build, and Deployment
Playwright End-to-End Tests

The test suite lives in tests/ and is configured via playwright.config.ts. Key settings: Chromium browser, baseURL at localhost:3000, sequential test runs (auth state is shared between tests), and a webServer block that auto-starts the Vite dev server before tests run.

The auth bypass is the most important test infrastructure decision: appending ?__test_mode=true to any URL causes withAuth.tsx to skip the Supabase auth check. This means all 31 tests run without needing live Supabase credentials in the CI environment.

Build Process

Running vite build produces a dist/ directory. Vite's rollupOptions in vite.config.ts apply manual chunk splitting: careers400Final.ts and bursaries.ts are isolated into separate chunks. React, motion/react, Supabase, and lucide-react are bundled into a vendor chunk. The result is that a student visiting the home page only downloads the app core and vendor chunk — the large career and bursary data files load only when those pages are visited.

Deployment

Every push to the GitHub main branch triggers an automatic Vercel deployment. Vercel runs vite build, deploys the dist/ output, and serves it globally. Environment variables stored in the Vercel project dashboard are injected at build time — VITE_GROQ_API_KEY for the AI assistant, plus the Supabase project URL and anon key.
