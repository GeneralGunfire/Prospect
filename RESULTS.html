Decision Records — What Changed and Why

Seven decisions during development caused the platform to change in a way that wasn't planned at the start. Each one was data-driven: a technical failure, a research finding, or a scope check against the original problem statement. The table below records what changed, why, and the measurable effect.

Decision What Changed Reason Measurable Effect Outcome
Framework switch Next.js replaced with React + Vite SSR added complexity the project didn't need. Vite's HMR is significantly faster for SPA development with large data files. Dev server cold start dropped from ~6s to ~0.8s. Build config reduced by ~60 lines. Revised
AI Backend 1: Gemini Google Gemini removed Hit the free-tier rate limit (429 error) on day one of live testing with normal student-level traffic. Unreliable for a free platform. Zero viable AI responses during the test window. Replaced within 24 hours. Removed
AI Backend 2: Ollama Ollama removed Ollama runs a local model server — it works on localhost only. Cannot function on a Vercel-hosted site where there is no persistent server process. Architecturally incompatible. Removed before production deployment. Removed
AI Backend 3: Groq Groq adopted as final AI backend Groq's cloud inference API is free-tier, fast (<3s typical), and OpenAI-compatible. Passed all timing benchmarks. llama-3.3-70b-versatile handles SA education queries accurately. Median response time 1.8s for text. 7 local instant answers added to reduce API calls. Kept
Pothole reporting removed Civic reporting feature dropped Research showed the target audience (Grade 8–12 students) is not the primary user of pothole reporting. Adds no value to a learner's study or career decisions. Scope reduced. Feature not built. Saves roughly 2–3 days development time. Removed
Community Impact page removed Community impact visualisation dropped During audience fit check, the proposed feature — showing community-level statistics — couldn't be reliably sourced or kept up-to-date. Data didn't exist in a usable form. Not built. Development effort reallocated to Study Library content. Removed
Code splitting added Vite rollupOptions: data files chunked separately Pre-split, the main JS bundle included careers400Final.ts and bursaries.ts inline, producing a >800KB initial load. Lighthouse flagged this as a performance issue. Initial JS payload dropped. Build time reduced from ~5.8s to ~2.1s after chunking. Revised

Prototype and Trial Progression
Platform Prototypes
P1
Next.js Scaffold
Week 1 · Revised
Initial scaffold using Next.js 14 with App Router. SSR added complexity without benefit — the platform has no SEO-critical dynamic routes, and server components introduced friction in the data loading pattern. Replaced after one week.
P2
React + Vite SPA (Core)
Week 2 · Kept
Switched to React 19 + Vite 6 + Tailwind CSS v4. Established routing with React Router v6, auth scaffolding with Supabase, and basic page shell. This became the foundation all subsequent work was built on.
P3
Career Explorer + Quiz
Weeks 3–4 · Kept
400 career profiles loaded from careers400Final.ts. Search, filter, and career detail pages functional. Career quiz multi-step flow implemented with ranking algorithm. First data-heavy prototype — revealed the bundle size issue later addressed with code splitting.
P4
Bursaries + Community Features
Weeks 4–5 · Partially revised
245 bursary listings added. Load shedding tracker, water dashboard, and NSC countdown integrated. Pothole reporting and community impact page initially prototyped here — both removed after audience fit analysis showed they didn't serve the learner use case.
P5
AI School Assistant + Study Library
Weeks 6–7 · Kept
Groq-powered chat interface implemented after Gemini and Ollama trials failed. 7 predefined instant answers added. Study library with 24 topics across 11 subjects added. Mastery tracking via Supabase implemented for two interactive algebra modules.
P6
Final Platform — Tested and Deployed
Week 8 · Launched
Code splitting applied to data chunks. 31 Playwright end-to-end tests written and passing. Data saver mode added. Deployed to Vercel with environment variables configured. Launched 15 May 2026.
AI Backend Trials
A1
Google Gemini
Trial 1 · Removed
Gemini's free tier hit its daily rate limit (HTTP 429) on day one of live testing. Under normal student-level usage — a few dozen requests per hour — the quota was exceeded before the end of a single test session. Not suitable for a free-to-access educational platform.
A2
Ollama (Local Model)
Trial 2 · Removed
Ollama runs a model inference server on the local machine. On localhost this works. On Vercel — where the site is hosted — there is no persistent local server, so Ollama's API is unreachable. Architecturally incompatible with serverless deployment. Removed before any production testing.
A3
Groq (llama-3.3-70b-versatile)
Trial 3 · Adopted
Groq's cloud inference API is free-tier, works on any hosted environment, and is OpenAI-compatible — meaning the integration code required minimal changes from the Gemini attempt. Median text response time 1.8 seconds in testing. Image requests (base64 upload) average 14 seconds — acceptable given the expected use case.
