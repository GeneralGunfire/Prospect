How the App Works — Page Routing
01
User clicks a link
No URL changes. onNavigate('page-name') is called.
02
App.tsx switches state
setPage('page-name') triggers a re-render.
03
Lazy component loads
Only that page's JS bundle downloads — fast on mobile.
04
withAuth checks session
Supabase session verified. Guest or redirect if not signed in.
Key Files Explained
App.tsx
The entire app lives here. Holds page state, lazy-imports every page component, and renders whichever one matches. No URL router — switching pages is just setPage().
lib/withAuth.tsx
Higher-order component that wraps every protected page. Checks Supabase session on mount. If no user, redirects to login. Also defines the AppPage type — every valid page name must be listed here.
pages/learning/…/TopicName.tsx
One file per study topic. Contains the full lesson: overview card, interactive lesson, guided practice with hints, scratchpad (Canvas 2D), quiz, and mastery tracking via Supabase. All self-contained.
services/schoolAssistService.ts
Sends messages to Groq's API (llama-3.3-70b-versatile) via fetch. Free inference — no Anthropic/OpenAI billing. Returns streamed responses with ~1.8s median latency.
data/careers400Final.ts
Static TypeScript array of 400+ career profiles — title, description, APS score, subjects required, salary range, and field. Filtered client-side. No database query needed.
data/bursaries.ts
245 bursary entries with funder, field of study, deadline, and application link. Filtered by subject interest and grade client-side. Updated manually each cycle.
