How Every Feature Works

All features explained under Prospect's three pillars — Career Guide, School Assist, and Community.

Pillar 1 Career Guide

Helps learners discover what they want to do, what their subjects allow them to do, and how to get there — including university, TVET, and bursary pathways.

RIASEC Career Quiz
pages/school/ · data/quizQuestions.ts · data/quizScoringLogic.ts
What it is

An interactive personality-based career quiz using the internationally recognised RIASEC framework (Realistic, Investigative, Artistic, Social, Enterprising, Conventional). Learners answer a series of questions about their interests and preferences, and the quiz produces a personalised career type profile that maps to real careers.

How it works
1
Questions are stored in a static TypeScript file — each question is tagged to one or more RIASEC dimensions.
2
As the learner answers, each response increments a score counter for the relevant dimension(s) in real time.
3
At completion, the scoring logic ranks the six dimensions by score and identifies the learner's top two or three — their RIASEC code (e.g. "ISA" or "REC").
4
The code is cross-referenced against the career database to surface the careers most aligned to that personality profile.
Languages and tech
TypeScript React 19 Client-side scoring (no server)
Career Explorer
data/careers400Final.ts · components/careers/ · services/mapService.ts
What it is

A searchable, filterable database of 400+ South African career profiles. Each career includes a title, field, description, required matric subjects, minimum APS score, salary range, job demand level, and which provinces the work is concentrated in. Learners can filter by subject, APS range, field, or search by keyword.

How it works
1
Data: all 400+ profiles live in a single TypeScript file. Each was researched and structured with AI assistance then reviewed for accuracy against South African context — SAQA qualifications, university admission requirements, and labour market data.
2
Filtering: all filtering happens entirely in the browser — no server query. The array is filtered by field, APS range, or subject in milliseconds.
3
Detail view: clicking a career opens a full profile with salary ranges, demand level by province, required subjects highlighted, and linked bursaries for that field.
4
Subject mapping: a separate subjectCareerMapping.ts file cross-references every matric subject to the careers it unlocks — powering the "what your subjects allow" view.
Languages and tech
TypeScript React 19 Client-side filtering 400+ static profiles
Bursary Finder
data/bursaries.ts · pages/school/
What it is

A searchable database of 245 South African bursaries covering government, corporate, and private funders across all major fields of study. Each entry has the funder name, field, deadline, eligibility criteria, and a direct application link. Free, no sign-up wall.

How it works
1
Data: 245 entries compiled and verified manually with AI assistance, cross-checked against official funder websites and NSFAS documentation.
2
Filtering: learners filter by field of study, province, or deadline — all client-side, instant results.
3
Career link: bursaries are tagged by field so when a learner views a career, relevant bursaries surface automatically in the same view.

Pillar 2 School Assist

Gives learners the tools to actually study and learn — an interactive curriculum-aligned library, an AI tutor, and a school calendar — regardless of which school they attend.

Interactive Study Library
pages/learning/ · data/studyLibrary/ · Supabase study_progress table
What it is

A free, curriculum-aligned library of 24 fully interactive study topics for Grade 10 learners across 8 subjects: Mathematics, Physical Sciences, Life Sciences, Accounting, Business Studies, Economics, CAT, and EGD. Each topic is a complete lesson: overview, interactive walkthrough, guided practice with hints, a scratchpad, a quiz, and mastery tracking.

Mapping school content and widening its coverage — and truly bringing information like this to life — matters. We are essentially building a meaningful, detailed, open-source, free library with everything a learner will ever need to study and understand their curriculum, levelling the playing field. We do not expect students to be taught by AI. We are trying to create a meaningful, step-by-step interactive experience so that every learner, regardless of their background or school, can access the best possible education.
How each topic was built
1
Content sourcing: textbook sections, CAPS curriculum outlines, and topic explanations were gathered and used as reference material alongside AI assistance to map the exact scope, sequence, and key concepts for each topic.
2
Topic mapping: each topic was structured into learning stages — overview, interactive lesson (concept by concept), guided practice (worked example with hints at each step), and an independent quiz. This mirrors how a good teacher structures a lesson.
3
Coding: every topic is a single self-contained TypeScript/React file. Interactive elements use React state machines — learners can only advance when they engage with each stage.
4
Scratchpad: a Canvas 2D drawing pad is built into every topic. Learners work through problems by hand — just like on exam paper. Work is saved to localStorage per topic and persists across sessions.
5
Mastery tracking: quiz results are written to a Supabase study_progress table. 2 out of 3 correct = mastered. Below that, the learner enters a remediation path before retrying.
Languages and tech
TypeScript React 19 Tailwind CSS Canvas 2D API Supabase PostgreSQL localStorage (scratchpad)
School Assist — AI Chat Tutor
pages/school/SchoolAssistChatPage.tsx · services/schoolAssistService.ts · Groq API
What it is

A free AI chat assistant where learners can get help with schoolwork, subject choices, APS scores, bursary applications, exam preparation, and career direction. It understands South African context — NSFAS, CAO, DBE, CAPS — and responds like a warm, knowledgeable school counsellor.

How it works
1
The learner types a message or attaches an image of their work or exam paper and presses send.
2
The app builds a message array — the last 10 messages for context plus the new message — and sends it to the Groq API via a standard fetch call.
3
The model used is llama-3.3-70b-versatile — a powerful open-source LLM. A custom system prompt configures it as "School Assist," an SA-context guidance counsellor.
4
The response arrives in ~1.8 seconds on average and streams into the chat interface in real time.
5
Chat history is stored in Supabase so returning learners can continue a previous conversation from any device.
Why Groq — not Claude or ChatGPT

Three AI backends were tested. Gemini was ruled out due to cost at scale. A local Ollama instance was incompatible with the Vercel deployment. Groq was adopted because it is free for reasonable usage, extremely fast (1.8s median), and runs a model that produces high-quality educational responses. The platform must always be free to learners — that constraint drove every infrastructure decision.

Languages and tech
TypeScript React 19 Groq API — llama-3.3-70b-versatile Supabase (chat history) Image upload — base64 encoding

Load Shedding Dashboard
services/loadshedding.ts · backend/loadshedding_scraper.py · GitHub Actions
What it is

A live dashboard showing the current national load shedding stage, a 7-day forecast, and province-level power alerts — transformer faults, cable faults, substation overloads, and planned maintenance. Refreshes automatically every 30 minutes.

How the scraper works
1
Language: Python 3.12, using requests (HTTP fetching) and BeautifulSoup (HTML parsing).
2
Source: the Eskom load shedding status page and their planned/unplanned outages page. The scraper reads the page HTML and extracts the current stage and any alert text.
3
Parsing: a keyword dictionary maps phrases like "stage 4", "suspended", "no load shedding" to numeric stage values. Fault keywords ("transformer", "cable fault", "substation") classify each alert type.
4
Output: the scraper writes a single JSON file — public/data/loadshedding/latest.json — containing the current stage, a forecast array, and active alerts.
5
Automation: GitHub Actions runs the script every 30 minutes. After it runs, Actions commits the updated JSON and pushes to the repo. Vercel picks up the push and redeploys in seconds.
6
Frontend: the React app fetches the JSON on load with a 10-minute client cache. If the fetch fails, a sensible fallback is shown (suspended since March 2024).
Languages and tech
Python 3.12 BeautifulSoup 4 requests GitHub Actions — every 30 min Static JSON on Vercel CDN
Water Dashboard
services/waterService.ts · backend/water_scraper.py · GitHub Actions · Supabase
What it is

A province-by-province water dashboard showing dam levels with trend indicators (rising, stable, falling), active outage alerts, current water restrictions, upcoming maintenance schedules, and a historical news section covering major SA water events such as the Cape Town Day Zero crisis and the 2023 Johannesburg water emergency.

How the scraper works
1
Language: Python 3.12, requests and BeautifulSoup — same stack as the load shedding scraper.
2
Dam data: scraped from the Department of Water and Sanitation's weekly dam levels page. The HTML table of dam names and capacity percentages is parsed, then each dam is mapped to its province using a hardcoded dictionary of all major South African dams.
3
Alert data: municipal water alert pages (Cape Town, eThekwini, Tshwane, Nelson Mandela Bay) are scraped for outage and restriction notices. Text is parsed for urgency keywords and affected area names.
4
Output: writes public/data/water/latest.json — structured by province, containing dams, alerts, restrictions, and maintenance entries.
5
Automation: GitHub Actions runs this daily at 07:00 SAST. Updated JSON is committed and pushed, triggering a Vercel redeploy.
6
Fallback chain: the frontend tries the static JSON first, then falls back to live Supabase queries, then falls back to curated seed data — so the dashboard always shows something useful even if scraping fails.
Languages and tech
Python 3.12 BeautifulSoup 4 GitHub Actions — daily 07:00 SAST Supabase (fallback) 3-layer fallback: JSON → Supabase → seed
